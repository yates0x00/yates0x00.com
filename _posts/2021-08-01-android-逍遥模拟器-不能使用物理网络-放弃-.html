目前， android 模拟器在国内有了不少品牌， 口碑不错的是逍遥模拟器。

海外仍然有genymotion,  必须是win8+ . 

有 android for x86. ： 使用vmware安装后，卡顿的无法使用。 

还有android SDK 下的虚拟机， 卡成鬼。

所以今天暂时用了一下 逍遥模拟器。 有如下体会： 

1. 相当流畅，媲美genymotion

2. 但是android 版本低， 只有 android 7, android 5, 4.x 

3. 可以一键换机， 更改imei, mac，品牌等。这个太有意思了。

4. 可以更改配置。 可以root 

下载

到官网下载即可。 https://www.xyaz.cn/

安装

一路next即可。 有2个软件： 一个是 模拟器，一个是多开器。  前者是单个的虚拟机，后者可以下载镜像， 实现多开。

记得：在64位机器上要下载 android 7.x 64位。 否则无法安装后续的xposed. 

使用（安装多个软件）

在 多开器 中，创建新的虚拟机（64位 android 7, 目前是只有7.1, 4.4, 5.1 三个版本）

启动 7.1 64位之后 : （如果你的CPU不是64位，那就貌似无解了。）

1. 修改网络配置。 需要在： 逍遥模拟器界面 -> 设置(注意不是安卓系统的设置） -> 网络 -> 网络模式 -> 详细设置， 然后就会从默认的NAT模式切换成你想要的 网卡 ，例如叫 qualcomm. 然后 点击详细设置， 会允许我们设置 是DHCP 还是静态。 选择DHCP， 然后 正常选择 burp . 

IP地址会在 android -> WLAN -> 右上角小齿轮 点击后 -> 配置WLAN， 可以看到IP地址和MAC地址。

这就可以以后正常使用burp 了。

2. 下载 xposed 64  位 for 逍遥，文章参考： https://bbs.xyaz.cn/forum.php?mod=viewthread&tid=372342

2.1 记得开启虚拟机的root

2.2 安装 RE文件管理器。 正常安全即可。 

2.3 设置好共享目录。 然后打开（模拟器 -> 右侧菜单 -> 共享  -> 设置 -> 重启， 然后继续回到 共享设置页面，点击安卓小图标， 就会在模拟器中打开该文件夹）

2.4 进入到RE文件夹，可以看到有2个tab.  左侧第1个是 / (也就是系统目录）, 右侧那个就是 2.3的共享目录了。

2.5 进入到共享目录， 找到对应的 64位文件夹解压缩之后的文件夹，名称  xposed ,  长按， 然后点击顶部的 “复制”图标

2.6 进入到 系统目录 ->  /system,  然后点击 粘贴图标。 （在X图标的上面）

2.7  记得重启。（先在模拟器中点击 软重启 ， 等屏幕黑掉之后，在 多开器中点击 关闭按钮， 再点击 启动按钮）

3. 重启后， 安装 终端模拟器。（注意：首次安装之后，可以运行，但是以后打开都会提示崩溃。所以遇到崩溃的话，卸载，重启，重新安装，珍惜重装后的第一次机会）

3.1  su root

3.2  cd /system/xposed

3.3  ./memu-script.sh 

然后注意读日志。 如果能正常安全，就可以了。 

如果出现 permission等报错， 那就多试几次 su root, whoami 

3.4 重启.  再进入 xpose,  发现框架可以使用了。  （出现绿色图标：xposed 框架89 版本已激活）

4. 安装 JustTrustMe.apk   

正常安装即可。 安装好之后， 进入到xposed 框架中 -> 模块 -> JustTrustMe , 勾选. 重启， 使之生效。

5. 安装cer证书文件

6. 打开 虚拟机 -> 设置 -> 网络 -> 网络模式 -> 不要选择默认模式NAT

仍然不行。

逍遥模拟器不能使用物理网络。 放弃。
